Regra de aplicação

Quando usar cada variação do HUB

  • HUB · Claude (laranja) — versão default. Hero, header, footer, peças onde a marca aparece sozinha sem âncora de seção.
  • HUB · contextual — quando o wordmark aparece dentro de uma seção dedicada a um sub-produto (ex: dentro da fold "Obsidian", o HUB no header daquela fold pega púrpura).
  • HUB · Tinta / Papel — papelaria oficial (cartão, envelope, contrato), assinatura de e-mail formal. Volta ao monocromático quando o contexto pede formalidade.
  • "Atinge" — nunca colorido. Sempre #F5F3EE (dark) ou #0A0A0A (light).

04Efeitos do sistema · biblioteca

5 padrões reutilizáveis · todos sem gradiente · all CSS

Cinco tratamentos que você pode chamar nas peças mobile e desktop. Todos respeitam a regra-mãe: sem gradiente, sem sombra colorida fora do laranja-Claude. O glow é drop-shadow ou radial-gradient de cor única para transparente — não conta como "gradiente colorido", é halo.

Observação técnica

Sobre os efeitos do site live (referência chaseai.io + sua landing)

Conferi seu landing/index.html + script.js. Os efeitos mobile aqui já refletem o vocabulário real: grid 48px, terminal hero, folds com glow radial, scrollspy lateral, infinity wheel. O que falta migrar pra mobile no site live, na minha leitura:

  • Brand-mark — a landing usa arcs+dot (provisório). Trocar pela montanha + bandeira laranja como SVG inline no .brand-mark dá identidade real ao header.
  • Wheel infinito — no mobile, hoje a SVG 700×340 escala mal. Sugestão: collapse pra lista vertical de chips em <640px (como o exemplo B acima), mantendo as 5 cores das ferramentas.
  • Folds com glow — já tem cor por fold (var --fold-accent), mas o glow radial não tá ativo no mobile. Adicionar ::before radial atrás do título dá pulso mesmo sem JS.
  • Scrollspy lateral — no mobile vira horizontal sticky no topo. Usar a cor de cada seção como dot ativo (em vez de só laranja) reforça o sistema cromático.
